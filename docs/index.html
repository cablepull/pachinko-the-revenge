Pachinko · The Revenge

an original revenge cabinet · CR-style · ST kakuhen

A single, deeply-realized digital pachinko cabinet — modern CR-style with ST kakuhen — in which an original revenge narrative is delivered through the reach hierarchy itself. The reach tiers map to plot beats: provocation, preparation, confrontation, catharsis. Players learn the plot by playing; the plot teaches them the reach taxonomy. Built in Rust + macroquad, compiled to WebAssembly.

  • Iteration 2  —  ball physics + pin field
    2026-05-25 · ~489 KB WASM · single machine, single arc, real playfield

    v0.1 looked like a slot machine — SPACE went directly to a reel spin. v0.2 adds the layer that makes pachinko pachinko: a launch knob (right side), a visible diamond-pattern pin field (~80 pins), steel balls with gravity and pin collisions, and a chucker (HESO) that is now the only path to a reel spin. The base game rate (~25–40 balls returned per 100 fired) is now an emergent property of the pin layout instead of a hidden constant. Driven by audit-001 (the ACH-shaped architecture audit produced via the intelligence-analyst skill after magnetfragnet's nudge-9 fired six iterations in a row).

The project

Authentic Japanese pachinko grammar (mixed-script overlays, brass-fanfare catharsis, shounen-anime cut-in animation, original-cast JP voice), but original IP. Not a parody, not a Westernized reinterpretation. The cabinet, sound design, parlor framing should pass a 常連 (regular) sniff test. 12 numbered constraints in intent.md, 23 numbered rules in the PRD citing those constraints, and a discipline-shaped feedback log on the tooling used to build it.
